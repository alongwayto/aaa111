## 1. 系统概述

**智能校园设备管理系统**是基于 SpringBoot + Vue 3 构建的全功能设备管理平台，涵盖：

- 设备全生命周期管理（录入、监控、维修、报废）
- 实时状态监控与异常预警
- 故障上报、派单、处理全流程跟踪
- 多维度数据统计与可视化报表
- AI 智能故障诊断与维护建议
- 完善的用户权限管理体系

**技术栈**：SpringBoot 2.7 / Vue 3 / MySQL 8.0 / MyBatis-Plus / Spring Security + JWT / Element Plus / ECharts

## 2. 快速开始

### 2.1 环境要求

| 组件 | 版本要求 |
|------|---------|
| JDK | 11+ |
| Node.js | 18+ |
| MySQL | 8.0+ |
| Maven | 3.6+ |

### 2.2 数据库初始化

```sql
-- 1. 创建数据库并执行建表脚本
mysql -u root -p < backend/src/main/resources/sql/schema.sql

-- 2. 导入初始化数据
mysql -u root -p campus_equipment < backend/src/main/resources/sql/data.sql
```

### 2.3 后端启动

```bash
cd backend

# 修改数据库配置（src/main/resources/application.yml）
# spring.datasource.username: 你的数据库用户名
# spring.datasource.password: 你的数据库密码

# 设置 OpenAI API Key（可选，用于 AI 功能）
export OPENAI_API_KEY=sk-your-api-key

# 启动
mvn spring-boot:run
```

后端启动后访问：
- API 接口：`http://localhost:8080/api`
- Swagger 文档：`http://localhost:8080/api/swagger-ui.html`

### 2.4 前端启动

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端访问地址：`http://localhost:5173`

## 3. 功能模块说明

### 3.1 首页运营驾驶舱

登录后默认进入运营驾驶舱，展示：

- **核心指标**：设备总数、在线率、平均健康度、未处理告警、待处理工单、近 6 月维护成本
- **健康分布**：按健康、关注、预警、高危展示设备状态
- **故障趋势**：展示近 7 天故障发生趋势
- **待办建议**：根据告警、风险设备和工单自动生成处理建议
- **风险设备**：展示健康评分较低的设备与维护建议
- **最新告警**：展示未处理告警，便于快速响应
- **工单与成本图表**：展示工单处理进度和维护成本变化

### 3.2 设备管理

#### 设备列表

- **搜索筛选**：支持按设备编号、名称、分类、状态、在线状态多条件筛选
- **新增设备**：填写设备基本信息（编号、名称、分类、位置、品牌、型号、购买信息等）
- **编辑设备**：修改设备信息
- **删除设备**：支持单条和批量删除
- **Excel 导入**：下载模板，填写后批量导入设备数据
- **Excel 导出**：将当前筛选结果导出为 Excel 文件
- **查看详情**：查看设备完整信息及历史状态记录图表

#### 设备分类

管理设备分类树形结构，支持多级分类（如：计算机设备 > 台式电脑）

#### 位置管理

管理设备安装位置，包含楼栋、楼层、房间信息

### 3.3 状态监控

#### 实时监控

- 展示所有在线设备的实时运行参数
- 监控指标：CPU 使用率、内存使用率、磁盘使用率、温度
- 进度条颜色：绿色（正常）、橙色（警告 >70%）、红色（危险 >90%）
- 实时监控大屏每 5 秒自动刷新

#### 预警管理

- 查看所有设备预警记录
- 预警级别：低（蓝）、中（橙）、高（红）、紧急（红）
- 预警类型：CPU 过高、内存过高、磁盘过高、温度过高、网络断开
- 支持处理预警（填写处理备注）或忽略预警

**预警阈值**：
| 指标 | 阈值 |
|------|------|
| CPU 使用率 | > 90% |
| 内存使用率 | > 85% |
| 磁盘使用率 | > 90% |
| 温度 | > 70℃ |

### 3.4 故障管理

#### 故障上报流程

```
用户上报故障 → 管理员派单 → 维护员开始处理 → 完成处理 → 归档
   (状态:待派单)  (状态:已派单)  (状态:处理中)   (状态:已完成) (状态:已归档)
```

**操作步骤**：

1. **上报故障**：选择故障设备，填写故障类型、级别和详细描述
2. **派单**（管理员）：选择维护员，设置预计完成时间
3. **开始处理**（维护员）：点击"开始处理"，记录实际开始时间
4. **完成工单**（维护员）：填写处理描述、结果、费用、更换配件
5. **归档**（管理员）：将已完成的故障记录归档

#### 工单管理

维护员可查看分配给自己的工单，进行处理操作

#### 故障归档

查看已完成和已归档的历史故障记录

### 3.5 数据统计

#### 使用率统计

- 设备在线率柱状图（按设备展示）
- 故障高频设备 TOP10 横向柱状图

#### 维护成本

- 近12个月维护费用月度趋势柱状图

#### 故障报表

## 5. 常见操作流程

### 5.1 新增设备完整流程

1. 进入**设备管理 > 设备分类**，确认分类已存在（如无则新增）
2. 进入**设备管理 > 位置管理**，确认位置已存在（如无则新增）
3. 进入**设备管理 > 设备列表**，点击"新增设备"
4. 填写设备编号（唯一）、名称、选择分类和位置
5. 填写品牌、型号、序列号、购买信息等
6. 点击"保存"完成创建

### 5.2 批量导入设备

1. 进入**设备管理 > 设备列表**，点击"导入"
2. 按照 Excel 模板格式填写设备数据：
   - 设备编号（必填，唯一）
   - 设备名称（必填）
   - 分类编码（需与系统中分类编码一致）
   - 其他字段按需填写
3. 上传文件，点击"开始导入"
4. 系统自动跳过已存在的设备编号

### 5.3 故障处理完整流程

**用户端**：
1. 发现设备故障，进入**故障管理 > 故障上报**
2. 点击"上报故障"，选择故障设备
3. 选择故障类型和级别，详细描述故障现象
4. 提交后等待管理员派单

**管理员端**：
1. 在**故障管理 > 故障上报**中看到新故障（状态：待派单）
2. 点击"派单"，选择合适的维护员，设置预计完成时间
3. 故障状态变为"已派单"

**维护员端**：
1. 在**故障管理 > 工单管理**中看到分配的工单
2. 点击"开始处理"，记录开始时间
3. 处理完成后点击"完成"，填写处理描述、费用等信息

**管理员归档**：
1. 在**故障管理 > 故障归档**中找到已完成的故障
2. 点击"归档"完成整个流程

## 6. AI 助手使用

### 6.1 故障诊断示例

**输入**：
```
我的设备出现了故障，请帮我诊断。
设备名称：联想台式电脑
故障现象：开机后显示器无信号，电源指示灯正常亮起，风扇运转正常
```

**AI 会给出**：
- 可能的故障原因分析
- 排查步骤（从简单到复杂）
- 解决方案建议

### 6.2 维护建议示例

**输入**：
```
请为以下设备提供预防性维护建议。
设备名称：戴尔服务器
设备类型：服务器
使用年限：3年
```

**AI 会给出**：
- 定期维护项目清单
- 维护频率建议
- 注意事项

### 6.3 注意事项

- AI 功能需要配置有效的 OpenAI API Key
- 如 API Key 未配置，AI 功能将返回错误提示
- 对话历史保留最近 10 条，可点击"清空对话"重新开始

## 7. 部署说明

### 7.1 生产环境部署

**后端打包**：
```bash
cd backend
mvn clean package -DskipTests
# 生成 target/equipment-management-1.0.0.jar
```

**启动命令**：
```bash
java -jar equipment-management-1.0.0.jar \
  --spring.datasource.url=jdbc:mysql://your-db-host:3306/campus_equipment \
  --spring.datasource.username=your-username \
  --spring.datasource.password=your-password \
  --OPENAI_API_KEY=sk-your-key
```

**前端打包**：
```bash
cd frontend
npm run build
# 生成 dist/ 目录，部署到 Nginx
```

**Nginx 配置示例**：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/campus-equipment/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 7.2 数据库备份恢复

**手动备份**：在系统管理 > 数据备份页面点击"立即备份"

**界面恢复**：在系统管理 > 数据备份页面选择成功备份，点击"恢复"并二次确认

**命令行备份**：
```bash
mysqldump -u root -p campus_equipment > backup_$(date +%Y%m%d).sql
```

**恢复**：
```bash
mysql -u root -p campus_equipment < backup_20240416.sql
```

## 8. 常见问题

**Q: 登录提示"用户名或密码错误"**
A: 确认使用正确的账号密码，默认密码为 `admin123`

**Q: 页面显示"未登录或登录已过期"**
A: Token 有效期为 24 小时，重新登录即可

**Q: AI 助手提示"服务暂时不可用"**
A: 检查后端配置中的 `OPENAI_API_KEY` 是否正确设置，或网络是否能访问 OpenAI API

**Q: Excel 导入失败**
A: 检查文件格式是否为 .xlsx 或 .xls，确认分类编码与系统中一致

**Q: 数据备份失败**
A: 确认服务器上已安装 `mysqldump` 命令，且 `backup.mysql-dump-path` 配置正确

**Q: 设备状态不更新**
A: 实时监控页面每 30 秒自动刷新，也可手动刷新页面

**Q: 如何修改预警阈值**
A: 当前版本阈值在后端代码 `MonitorServiceImpl.java` 中配置，后续版本将支持界面配置
