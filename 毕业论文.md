# 基于SpringBoot+Vue的智能校园设备管理系统的设计与实现









## 摘要



随着教育信息化2.0战略的深入推进，高校校园设备数量与种类持续增长，传统管理模式存在信息零散、流程低效、数据价值未挖掘、权限管控缺失等突出问题，已难以满足现代化管理需求。本研究设计并实现了一套基于SpringBoot+Vue的智能校园设备管理系统，采用前后端分离的B/S架构，后端基于SpringBoot 2.7框架，集成Spring Security实现RBAC权限控制与JWT无状态认证，采用MyBatis-Plus进行数据持久化操作；前端基于Vue 3框架，搭配Element Plus组件库与ECharts可视化引擎，构建了响应式交互界面。系统涵盖设备信息管理、状态监控与预警、故障处理与工单流转、数据统计分析、系统运维管理五大核心模块，实现了设备全生命周期的闭环管理。系统支持设备数据批量导入导出、多角色权限隔离、操作日志审计、数据库定时备份与恢复等功能，保障了数据安全与业务可追溯性。测试结果表明，系统功能完整、运行稳定，核心接口响应时间均低于500ms，满足高校设备管理的实际需求。



**关键词**：校园设备管理；SpringBoot；Vue；前后端分离；RBAC；状态监控



---



## Abstract



With the advancement of the Education Informatization 2.0 strategy, the number and variety of campus equipment in universities continue to grow. Traditional management models suffer from scattered information, inefficient processes, unexploited data value, and lack of access control, which can no longer meet modern management needs. This study designs and implements an intelligent campus equipment management system based on SpringBoot+Vue, adopting a front-end and back-end separated B/S architecture. The back-end is built on the SpringBoot 2.7 framework, integrating Spring Security for RBAC access control and JWT stateless authentication, and using MyBatis-Plus for data persistence. The front-end is based on the Vue 3 framework, combined with the Element Plus component library and ECharts visualization engine to build a responsive interactive interface. The system covers five core modules: equipment information management, status monitoring and early warning, fault handling and work order flow, data statistical analysis, and system operation and maintenance management, achieving closed-loop management of the equipment lifecycle. The system supports batch import and export of equipment data, multi-role access isolation, operation log auditing, scheduled database backup and recovery, ensuring data security and business traceability. Test results show that the system has complete functionality and stable operation, with core API response times below 500ms, meeting the practical needs of university equipment management.



**Keywords**: Campus Equipment Management; SpringBoot; Vue; Front-end and Back-end Separation; RBAC; Status Monitoring

## 第1章 绪论



### 1.1 研究背景与意义



#### 1.1.1 研究背景



随着教育信息化2.0战略的深入推进，高校校园设备（如教学设备、实验仪器、后勤设施等）的数量与种类持续增长，设备管理的复杂度呈指数级上升。据教育部统计数据显示，全国高校教学科研仪器设备总值已超过万亿元，且每年以10%以上的速度增长。然而，当前多数高校仍采用传统管理模式，存在以下突出问题：



（1）信息管理零散化。设备信息分散在多个Excel表格或纸质台账中，不同部门之间的数据难以互通，导致信息不一致、查询效率低下，无法形成统一的设备资产视图。



（2）业务流程低效化。故障上报、维修调度等关键业务流程依赖人工传递，从故障发现到维修响应往往需要数天时间，严重影响教学和科研工作的正常开展。



（3）数据价值未挖掘。大量设备运行数据、维护记录、故障信息缺乏有效的分析手段，无法为设备采购决策、维护策略优化、资产折旧评估等提供数据支撑。



（4）权限管控缺失。传统系统缺乏细粒度的权限控制机制，不同角色（管理员、维护人员、普通教职工）的操作边界模糊，数据安全存在隐患。



在此背景下，SpringBoot+Vue作为前后端分离架构的主流技术组合，凭借SpringBoot的高效开发能力、稳定的后端支撑，以及Vue的组件化交互、响应式布局优势，为构建轻量化、高可用的校园设备管理系统提供了成熟的技术底座。开发一套适配高校场景的智能设备管理系统，成为解决传统管理痛点、推动校园后勤数字化转型的必然需求。



#### 1.1.2 研究意义



本研究的设计与实现具有以下理论意义与实践价值：



（1）完善设备全生命周期管理的设计逻辑。结合高校设备管理的特殊性，构建"信息录入-状态监控-故障处理-维护归档-数据统计"的闭环管理模型，为校园资产数字化管理提供理论与设计支撑。



（2）强化轻量级系统的安全性设计。实践"前端交互验证+后端兜底校验+角色权限隔离"的三重安全机制，为中小型Web系统的安全设计提供可复用方案。



（3）提升校园设备管理效率。系统通过批量导入导出、故障一键上报、维护自动调度等功能，减少人工操作成本，缩短业务响应时间。



（4）保障设备运维的连续性与可追溯性。系统完整记录设备采购信息、维护记录、故障处理流程，实现全生命周期数据追溯，为设备折旧评估、报废决策提供可靠依据。



（5）具备良好的推广价值。系统采用轻量化设计，可根据不同高校的规模与需求灵活调整功能模块，适配本科院校、职业院校等不同类型高校的设备管理需求。



### 1.2 国内外研究现状



#### 1.2.1 国外研究现状



在欧美发达国家，高校设备管理信息化起步较早，已形成较为成熟的解决方案。美国高校普遍采用资产管理信息系统（如SchoolDude、AssetWorks等），实现了设备采购、维护、报废的全流程数字化管理。这些系统通常集成了物联网传感器，能够实时采集设备运行参数，实现预测性维护。英国高校则倾向于使用基于ITIL框架的服务管理系统，将设备管理纳入IT服务管理的整体框架中，强调服务级别协议（SLA）和持续改进。



日本高校在设备管理方面注重精细化和标准化，普遍采用条码/RFID技术进行设备标识和追踪，结合移动终端实现巡检和盘点自动化。澳大利亚高校则强调设备共享和利用率优化，通过在线预约系统提高大型仪器的使用效率。



然而，国外商业系统通常价格昂贵、定制化程度低，且业务流程与国内高校的管理模式存在差异，难以直接移植使用。



#### 1.2.2 国内研究现状



近年来，国内高校设备管理信息化建设取得了显著进展。清华大学、浙江大学等"双一流"高校已建立了较为完善的设备资产管理平台，实现了设备全生命周期管理。但在地方本科院校和职业院校中，设备管理信息化水平参差不齐，仍存在以下不足：



（1）系统功能单一。多数系统仅实现了设备台账管理，缺乏状态监控、故障处理、数据分析等高级功能。



（2）技术架构陈旧。部分系统仍采用JSP+Servlet的传统架构，前后端耦合严重，维护困难，扩展性差。



（3）用户体验欠佳。界面设计落后，操作流程繁琐，缺乏数据可视化能力，用户接受度低。



（4）安全机制薄弱。缺乏统一的认证授权机制，权限控制粒度粗，操作审计不完善。



随着SpringBoot和Vue等现代Web开发技术的成熟，基于前后端分离架构开发轻量化、高可用的设备管理系统已成为主流趋势。本研究正是在此背景下，结合高校设备管理的实际需求，设计并实现一套功能完善、安全可靠的智能校园设备管理系统。

## 第2章 相关技术概述



### 2.1 SpringBoot框架



SpringBoot是由Pivotal团队开发的基于Spring框架的快速开发脚手架，其核心设计理念是"约定优于配置"。SpringBoot通过自动配置机制，使开发者无需编写大量的XML配置文件即可快速搭建基于Spring的应用程序。本系统采用SpringBoot 2.7.18版本，主要利用了以下特性：



（1）自动配置。SpringBoot根据项目中引入的依赖自动配置相应的Bean，如数据源、事务管理器、视图解析器等，大幅减少了配置工作量。



（2）起步依赖。通过spring-boot-starter-web、spring-boot-starter-security等起步依赖，简化了Maven依赖管理，避免了版本冲突问题。



（3）内嵌服务器。默认内嵌Tomcat服务器，无需部署WAR包到外部应用服务器，通过java -jar命令即可直接运行。



（4）Actuator监控。提供应用健康检查、运行指标收集等生产级监控能力。



### 2.2 Vue框架与Element Plus



Vue.js是一套用于构建用户界面的渐进式JavaScript框架，其核心特性包括响应式数据绑定和组件化开发。本系统前端采用Vue 3.4版本，主要利用了以下特性：



（1）Composition API。Vue 3引入的组合式API（setup语法糖）使代码组织更加灵活，逻辑复用更加便捷，特别适合复杂业务组件的开发。



（2）响应式系统。基于Proxy的响应式系统，能够精确追踪依赖关系，实现细粒度的更新优化。



（3）组件化开发。将页面拆分为可复用的组件，每个组件封装自己的模板、逻辑和样式，提高了代码的可维护性。



Element Plus是基于Vue 3的桌面端UI组件库，提供了表单、表格、对话框、分页等60余种高质量组件，本系统采用Element Plus 2.4.4版本，用于快速构建统一风格的管理界面。



### 2.3 MyBatis-Plus



MyBatis-Plus是MyBatis的增强工具，在MyBatis的基础上只做增强不做改变，旨在简化开发、提高效率。本系统采用MyBatis-Plus 3.5.5版本，主要利用了以下特性：



（1）通用CRUD。内置通用的Mapper和Service，无需编写XML即可完成单表的CRUD操作，大幅减少样板代码。



（2）条件构造器。提供LambdaQueryWrapper和LambdaUpdateWrapper，以Lambda表达式的方式构建查询条件，避免了硬编码字段名。



（3）分页插件。内置PaginationInnerInterceptor分页插件，通过简单的配置即可实现物理分页。



（4）逻辑删除。通过@TableLogic注解和全局配置，实现数据的逻辑删除而非物理删除，保障数据安全。



（5）自动填充。通过@TableField注解和MetaObjectHandler，实现创建时间、更新时间等字段的自动填充。



### 2.4 Spring Security与JWT



Spring Security是Spring生态中的安全框架，提供了认证、授权、攻击防护等全面的安全服务。本系统采用Spring Security实现用户认证与授权，核心特性包括：



（1）认证管理。通过AuthenticationManager和UserDetailsService实现基于用户名密码的身份认证。



（2）方法级授权。通过@EnableGlobalMethodSecurity和@PreAuthorize注解，实现细粒度的方法级权限控制。



（3）过滤器链。通过自定义JwtAuthenticationFilter，实现基于JWT Token的无状态认证。



JWT（JSON Web Token）是一种开放标准（RFC 7519）的无状态认证方案。JWT将用户信息编码为三段式Token（Header.Payload.Signature），服务端无需存储Session即可验证用户身份。本系统采用JJWT 0.11.5库生成和解析JWT Token，Token有效期为24小时，签名算法为HS256。



### 2.5 ECharts数据可视化



ECharts是百度开源的基于JavaScript的数据可视化图表库，提供了折线图、柱状图、饼图、仪表盘等30余种图表类型。本系统采用ECharts 5.4.3版本，通过vue-echarts 6.6.8封装组件，实现了设备状态分布饼图、故障趋势折线图、维护成本柱状图、设备在线率仪表盘等可视化图表。



### 2.6 EasyExcel



EasyExcel是阿里巴巴开源的基于Java的Excel读写工具，针对大文件读写进行了内存优化，避免了POI的内存溢出问题。本系统采用EasyExcel 3.3.3版本，实现了设备数据的批量导入导出功能，支持自定义字段映射和数据校验。



### 2.7 本章小结



本章对系统开发所采用的关键技术进行了概述，包括后端框架SpringBoot、前端框架Vue与Element Plus、ORM框架MyBatis-Plus、安全框架Spring Security与JWT、可视化引擎ECharts以及Excel处理工具EasyExcel。这些技术的合理选型与组合，为系统的高效开发与稳定运行提供了坚实的技术基础。

### 3.1 系统概述



智能校园设备管理系统面向高校设备管理中心，旨在为管理员、维护员和普通教职工三类用户提供统一的设备管理平台。系统覆盖设备从采购入库到报废处置的全生命周期，实现设备信息管理、运行状态监控、故障处理与工单流转、数据统计分析、系统运维管理五大核心业务。

### 3.3 角色权限需求分析



系统设计三类角色，采用RBAC（Role-Based Access Control）权限模型，各角色的权限范围如表3-6所示。



表3-6 角色权限分配表



| 角色 | 角色编码 | 功能权限范围 | 代表用户 |
| :--- | :--- | :--- | :--- |
| 超级管理员 | ROLE_ADMIN | 系统全部功能权限，包括用户管理、角色管理、日志管理、数据备份恢复、故障派单与归档 | 设备管理中心负责人 |
| 维护员 | ROLE_MAINTAINER | 首页、设备管理（增删改查+导入导出）、状态监控、故障处理、数据统计 | 设备维护人员 |
| 普通用户 | ROLE_USER | 首页、设备查询、实时监控、故障上报 | 教职工、学生 |



系统采用"用户-角色-权限"三级RBAC模型，用户与角色为多对多关系，角色与权限为多对多关系。权限分为菜单权限、按钮权限和接口权限三种类型，通过权限编码（如system:user、device:list等）进行细粒度控制。



### 3.4 非功能需求分析



#### 3.4.1 性能需求



（1）核心接口响应时间不超过500ms。



（2）支持100个并发用户同时在线操作。



（3）页面首次加载时间不超过3秒。



#### 3.4.2 安全性需求



（1）用户密码采用BCrypt算法加密存储，禁止明文存储。



（2）采用JWT Token进行无状态认证，Token有效期为24小时。



（3）所有写操作需进行权限校验，防止越权访问。



（4）关键操作需记录操作日志，包括操作人、操作时间、操作内容、客户端IP等。



（5）防止SQL注入、XSS攻击等常见Web安全威胁。



#### 3.4.3 兼容性需求



（1）服务端支持Windows和Linux操作系统部署。



（2）客户端兼容Chrome、Firefox、Safari、Edge等主流浏览器。



（3）前端界面支持1920×1080和1366×768等常见分辨率。



#### 3.4.4 可扩展性需求



（1）系统架构设计支持功能模块的横向扩展。



（2）数据库设计遵循三范式，便于后续功能扩展。



（3）接口设计遵循RESTful规范，便于第三方系统集成。



### 3.5 本章小结



本章从功能需求、角色权限需求和非功能需求三个维度对系统进行了全面的需求分析。功能需求方面，明确了设备信息管理、状态监控与预警、故障管理、数据统计分析、系统管理五大模块共42项功能需求。角色权限方面，设计了管理员、维护员、普通用户三类角色的权限体系。非功能需求方面，从性能、安全性、兼容性、可扩展性四个方面提出了明确的要求。这些需求分析结果为后续的系统设计与实现奠定了基础。

### 4.4 安全设计



#### 4.4.1 JWT认证流程



系统采用JWT无状态认证方案，认证流程如下：



（1）用户提交用户名和密码，调用登录接口。



（2）后端通过AuthenticationManager验证用户名密码。



（3）验证成功后，JwtUtils生成JWT Token（HS256签名，包含username和userId，有效期24小时）。



（4）前端将Token存储在localStorage中，后续请求通过Authorization: Bearer \<token\>头传递。



（5）JwtAuthenticationFilter从请求头提取Token，验证有效性，加载UserDetails设置到SecurityContext。



（6）Token过期返回401状态码，前端拦截后清除登录状态并跳转登录页。



#### 4.4.2 RBAC权限控制



系统采用"用户-角色-权限"三级RBAC模型：



（1）用户与角色为多对多关系，通过sys_user_role关联表实现。



（2）角色与权限为多对多关系，通过sys_role_permission关联表实现。



（3）权限分为菜单权限（permType=1）、按钮权限（permType=2）、接口权限（permType=3）三种类型。



（4）后端通过@PreAuthorize注解进行方法级权限控制，支持角色编码（如ROLE_ADMIN）和权限编码（如system:user）两种控制方式。



（5）前端通过userStore.hasPermission()方法进行菜单和按钮级别的权限控制，ROLE_ADMIN角色默认拥有所有权限。



#### 4.4.3 数据安全



（1）密码安全：用户密码采用BCrypt算法加密存储，强度因子为10。



（2）逻辑删除：核心业务表采用逻辑删除策略，deleted字段标记删除状态，数据不可见但保留在数据库中。



（3）输入校验：前后端双重校验，前端通过Element Plus表单验证，后端通过@Valid注解和自定义校验器。



（4）SQL注入防护：MyBatis-Plus的参数化查询机制有效防止SQL注入攻击。



### 4.5 本章小结



本章从系统架构设计、数据库设计、接口设计和安全设计四个方面完成了系统的整体设计。架构设计方面，采用前后端分离的B/S架构，明确了前端展示层、后端服务层和数据存储层的职责划分。数据库设计方面，设计了15张数据表，建立了用户-角色-权限的RBAC模型和设备-故障-工单的业务关联。接口设计方面，制定了RESTful接口规范，设计了40余个API接口。安全设计方面，实现了JWT认证、RBAC权限控制和多层次的数据安全策略。

#### 5.2.4 JWT认证过滤器



JwtAuthenticationFilter继承OncePerRequestFilter，实现JWT Token的提取与验证：



```java
@Component
@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    private final JwtUtils jwtUtils;
    private final UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        String token = resolveToken(request);
        if (StringUtils.hasText(token) && jwtUtils.validateToken(token)) {
            String username = jwtUtils.getUsernameFromToken(token);
            UserDetails userDetails = userDetailsService.loadUserByUsername(username);
            UsernamePasswordAuthenticationToken authentication =
                    new UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities());
            SecurityContextHolder.getContext().setAuthentication(authentication);
        }
        filterChain.doFilter(request, response);
    }

    private String resolveToken(HttpServletRequest request) {
        String bearerToken = request.getHeader("Authorization");
        if (StringUtils.hasText(bearerToken) && bearerToken.startsWith("Bearer ")) {
            return bearerToken.substring(7);
        }
        return null;
    }
}
```



#### 5.2.5 安全配置



SecurityConfig配置Spring Security的安全策略：



```java
@Configuration
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class SecurityConfig {

    @Autowired
    private JwtAuthenticationFilter jwtAuthenticationFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeRequests()
            .antMatchers("/auth/login", "/auth/register").permitAll()
            .antMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
            .antMatchers(HttpMethod.OPTIONS, "/**").permitAll()
            .anyRequest().authenticated()
            .and()
            .exceptionHandling()
            .authenticationEntryPoint((req, resp, ex) -> {
                resp.setContentType("application/json;charset=UTF-8");
                resp.setStatus(401);
                resp.getWriter().write("{\"code\":401,\"message\":\"未登录或登录已过期\"}");
            })
            .accessDeniedHandler((req, resp, ex) -> {
                resp.setContentType("application/json;charset=UTF-8");
                resp.setStatus(403);
                resp.getWriter().write("{\"code\":403,\"message\":\"无权限访问\"}");
            })
            .and()
            .addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

```java
    @Transactional
    public void importDevices(MultipartFile file) throws IOException {
        List<DeviceExcelModel> list = EasyExcel.read(file.getInputStream())
                .head(DeviceExcelModel.class).sheet().doReadSync();
        Map<String, Long> categoryCodeMap = categoryMapper.selectList(null).stream()
                .collect(Collectors.toMap(DeviceCategory::getCode, DeviceCategory::getId));
        for (DeviceExcelModel model : list) {
            if (deviceMapper.selectCount(new LambdaQueryWrapper<DeviceInfo>()
                    .eq(DeviceInfo::getDeviceCode, model.getDeviceCode())) > 0) {
                continue;
            }
            DeviceInfo device = new DeviceInfo();
            BeanUtils.copyProperties(model, device);
            if (StringUtils.isNotBlank(model.getCategoryCode())) {
                Long categoryId = categoryCodeMap.get(model.getCategoryCode());
                device.setCategoryId(categoryId);
            }
            if (StringUtils.isNotBlank(model.getBuilding())) {
                DeviceLocation location = getOrCreateLocation(model);
                device.setLocationId(location.getId());
            }
            deviceMapper.insert(device);
        }
    }
}
```

```java
    @Override

            throw new BusinessException("故障记录不存在");
        }
        if (fault.getStatus() != 0) {
            throw new BusinessException("故障已被处理");
        }

        order.setOrderNo(generateOrderNo());
        order.setFaultId(dto.getFaultId());

        order.setAssignerName(operatorName);
        order.setAssignTime(LocalDateTime.now());
        order.setExpectedTime(dto.getExpectedTime());
        order.setStatus(0);
        workOrderMapper.insert(order);

        fault.setStatus(1);
        faultMapper.updateById(fault);
        return order;
    }

    @Override
    @Transactional
    public void completeHandle(WorkOrderHandleDTO dto) {
        FaultWorkOrder order = workOrderMapper.selectById(dto.getOrderId());
        if (order == null) {
            throw new BusinessException("工单不存在");
        }
        order.setHandleDesc(dto.getHandleDesc());
        order.setHandleResult(dto.getHandleResult());
        order.setCost(dto.getCost());
        order.setPartsReplaced(dto.getPartsReplaced());
        order.setActualEnd(LocalDateTime.now());
        order.setStatus(2);
        workOrderMapper.updateById(order);

        FaultReport fault = faultMapper.selectById(order.getFaultId());
        if (fault != null) {
            fault.setStatus(3);
            faultMapper.updateById(fault);
        }

        if (dto.getCost() != null && dto.getCost().compareTo(BigDecimal.ZERO) > 0) {
            syncMaintenanceCost(order, fault);
        }
    }
```

```java
            createAlertIfNotExists(device, "cpu", 3, record.getCpuUsage(), 90,
                    device.getDeviceName() + " CPU使用率超过90%");
        }
        if (record.getMemoryUsage() != null && record.getMemoryUsage() > 85) {
            createAlertIfNotExists(device, "memory", 2, record.getMemoryUsage(), 85,
                    device.getDeviceName() + " 内存使用率超过85%");
        }
        if (record.getDiskUsage() != null && record.getDiskUsage() > 90) {
            createAlertIfNotExists(device, "disk", 3, record.getDiskUsage(), 90,
                    device.getDeviceName() + " 磁盘使用率超过90%");
        }
        if (record.getTemperature() != null && record.getTemperature() > 70) {
            createAlertIfNotExists(device, "temperature", 3, record.getTemperature(), 70,
                    device.getDeviceName() + " 温度超过70℃");
        }
        if (record.getNetworkStatus() != null && record.getNetworkStatus() == 0) {
            createAlertIfNotExists(device, "offline", 4, null, null,
                    device.getDeviceName() + " 网络离线");
        }
    }

    private void createAlertIfNotExists(DeviceInfo device, String alertType,
                                         int alertLevel, BigDecimal alertValue,
                                         BigDecimal threshold, String alertMsg) {
        Long count = alertMapper.selectCount(
            new LambdaQueryWrapper<DeviceAlert>()
                .eq(DeviceAlert::getDeviceId, device.getId())
                .eq(DeviceAlert::getAlertType, alertType)
                .eq(DeviceAlert::getStatus, 0));
        if (count > 0) return;

        DeviceAlert alert = new DeviceAlert();
        alert.setDeviceId(device.getId());
        alert.setDeviceCode(device.getDeviceCode());
        alert.setDeviceName(device.getDeviceName());
        alert.setAlertType(alertType);
        alert.setAlertLevel(alertLevel);
        alert.setAlertValue(alertValue);
        alert.setThreshold(threshold);
        alert.setAlertMsg(alertMsg);
        alertMapper.insert(alert);
    }
```

#### 5.2.9 操作日志AOP实现



通过自定义注解@OperationLog和AOP切面，实现关键操作的自动日志记录：



```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface OperationLog {
    String module();
    String operation();
    boolean saveParams() default true;
    boolean saveResult() default false;
}

@Aspect
@Component
@RequiredArgsConstructor
public class OperationLogAspect {

    private final SysOperationLogMapper logMapper;

    @Around("@annotation(operationLog)")
    public Object around(ProceedingJoinPoint point, OperationLog operationLog) throws Throwable {
        SysOperationLog log = new SysOperationLog();
        log.setModule(operationLog.module());
        log.setOperation(operationLog.operation());
        log.setMethod(point.getSignature().getDeclaringTypeName() + "." + point.getSignature().getName());
        log.setRequestUrl(request.getRequestURI());
        log.setRequestMethod(request.getMethod());
        log.setIp(getClientIp(request));
        log.setStartTime(System.currentTimeMillis());

        if (operationLog.saveParams()) {
            log.setRequestParams(JSON.toJSONString(point.getArgs()));
        }

        try {
            Object result = point.proceed();
            log.setStatus(1);
            if (operationLog.saveResult()) {
                log.setResponseData(JSON.toJSONString(result));
            }
            return result;
        } catch (Exception e) {
            log.setStatus(0);
            log.setErrorMsg(e.getMessage());
            throw e;
        } finally {
            log.setCostTime(System.currentTimeMillis() - log.getStartTime());
            logMapper.insert(log);
        }
    }
}
```

```javascript
        },
```

```javascript
            this.roles = []

        }
```

## 第7章 总结与展望



### 7.1 研究总结



本研究设计并实现了一套基于SpringBoot+Vue的智能校园设备管理系统，主要研究成果如下：



（1）完成了系统整体架构设计。采用前后端分离的B/S架构，后端基于SpringBoot 2.7框架，前端基于Vue 3框架，实现了设备管理、状态监控与预警、故障处理与工单流转、数据统计分析、系统运维管理五大核心模块，共42项功能需求。



（2）实现了设备全生命周期管理。支持设备信息的录入、查询、修改、删除，以及Excel批量导入导出功能，覆盖设备从采购入库到报废处置的完整生命周期。



（3）实现了多角色权限控制。基于RBAC模型，设计了管理员、维护员、普通用户三类角色，实现了"用户-角色-权限"三级权限体系，支持菜单权限、按钮权限和接口权限三种粒度的权限控制。



（4）实现了设备状态监控与自动预警。支持设备运行参数的实时采集，自动检测CPU、内存、磁盘、温度、网络等异常指标并生成告警，实现了设备健康度评估和风险等级划分。



（5）实现了故障处理全流程管理。支持故障上报、派单、处理、归档的完整业务流程，自动生成故障单号和工单编号，工单完成时自动同步维护成本。



（6）实现了数据可视化分析。集成ECharts图表组件，实现了运营驾驶舱、设备状态分布、故障趋势分析、维护成本统计、设备使用率等多维度的数据可视化展示。



（7）保障了系统安全性与可追溯性。实现了JWT无状态认证、BCrypt密码加密、全局异常处理、操作日志审计、数据库定时备份与恢复等安全机制。



### 7.2 研究不足



本研究仍存在以下不足之处：



（1）移动端适配不够完善。当前系统主要针对PC端设计，在移动端（手机、平板）的显示效果和交互体验有待优化。



（2）实时消息推送功能缺失。设备告警和工单状态变更目前采用轮询方式获取，未实现基于WebSocket的实时推送，信息及时性有待提升。



（3）设备硬件接口未对接。当前设备状态数据通过定时模拟生成，未实现与实际设备硬件（如物联网传感器）的接口对接，实时性不足。



（4）测试覆盖不够全面。单元测试覆盖率较低，主要依赖手工功能测试，自动化测试体系有待完善。



### 7.3 未来展望



针对上述不足，未来可从以下方向进行改进和扩展：



（1）移动端优化。开发响应式移动端适配版本或独立的移动端App，支持手机端随时随地查看设备状态、上报故障、接收告警推送。



（2）实时消息推送。集成WebSocket技术，实现设备告警和工单状态变更的实时推送，提升信息传递的及时性。



（3）物联网对接。扩展设备硬件接口，通过MQTT、CoAP等物联网协议对接实际设备传感器，实现设备状态的自动采集和远程控制。



（4）自动化测试完善。引入JUnit 5单元测试、Selenium端到端测试、JMeter自动化性能测试，建立完善的自动化测试体系。



（5）微服务化改造。随着系统规模扩大，可将单体架构拆分为用户服务、设备服务、故障服务、监控服务等微服务，提升系统的可扩展性和可维护性。
